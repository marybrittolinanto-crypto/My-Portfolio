Hospital Management System Using SQL

1. Introduction

The Hospital Management System is a database management project developed using SQL (Structured Query Language) to manage and organize hospital-related information efficiently. The main purpose of this project is to reduce manual work, improve data accuracy, and maintain hospital records in an organized digital format.

The system helps manage patient details, doctor schedules, room allocation, appointment records, medicines, billing details, and staff information efficiently using SQL database management.

2. Objective of the Project

The main objective of this project is to create an efficient database system for managing hospital operations using SQL.

  • To maintain patient information systematically
  • To manage doctor and department details
  • To schedule appointments
  • To maintain room allocation details
  • To manage billing information
  • To reduce manual paperwork
  • To implement advanced SQL concepts

3. Database Tables Used

  • Departments – Stores department details
  • Doctors – Stores doctor information
  • Patients – Maintains patient details
  • Nurses – Stores nurse information
  • Rooms – Maintains room allocation details
  • Appointments – Stores appointment details
  • Admissions – Manages patient admissions
  • Medicines – Stores medicine details
  • Bills – Maintains billing details
  • Staff – Stores staff details
  • Users – Stores login information

4. SQL Concepts Used

4.1 JOIN Operations

JOIN operations were used to connect multiple tables and retrieve related information efficiently. Doctor and Department JOIN was used to display doctor specialization and department details. Patient and Doctor Appointment JOIN was used to retrieve appointment information.

4.2 Aggregate Function

The SUM() function was used to calculate the total hospital income from bills.

4.3 View

A View named hospital_view was created to simplify appointment-related data retrieval.

4.4 Stored Procedure

A stored procedure named patient_details() was created to display patient details including name, age, and blood group.

4.5 Function

A function named room_tax() was created to calculate 5% tax on room charges.

4.6 Trigger

A trigger named patient_insert_trigger was implemented to automatically store new patient records in the audit log table.

5. Advantages of the System

  • Organized hospital record management
  • Reduced paperwork
  • Faster patient information retrieval
  • Better appointment scheduling
  • Efficient billing management
  • Secure database handling
  • Automation using SQL triggers and procedures

6. Conclusion

The Hospital Management System using SQL demonstrates how database management techniques can improve hospital operations. The project implements advanced SQL concepts such as JOIN, Views, Stored Procedures, Functions, and Triggers to improve efficiency and automation.
